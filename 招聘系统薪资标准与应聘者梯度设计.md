# 招聘系统薪资标准与应聘者梯度设计

## 概述
本次调整优化了招聘系统的薪资标准和应聘者数量机制，确保玩家必须支付合理薪资才能吸引到足够的应聘者。

---

## 核心规则

### 1. 最低薪资标准（硬性要求）
每个技能等级都有对应的最低薪资标准，**不允许低于此标准发布岗位**：

| 技能等级 | 最低薪资标准 |
|---------|-------------|
| Lv.1    | ¥10,000     |
| Lv.2    | ¥20,000     |
| Lv.3    | ¥30,000     |
| Lv.4    | ¥40,000     |
| Lv.5    | ¥50,000     |

**计算公式**：最低薪资 = 技能等级 × 10,000元

---

## 应聘者数量梯度

### 2. 薪资比率与吸引力

根据薪资与最低标准的比率，岗位吸引力呈梯度增长：

| 薪资比率 | 吸引力值 | 应聘者数量 | UI提示 |
|---------|---------|-----------|--------|
| 100% (刚达标) | 0.15 | 极少 (1-2人/天) | ⚠️ 刚达标准 - 极少应聘者，建议提高薪资！ |
| 105% (+5%) | 0.3 | 少量 (2-3人/天) | 📝 略高于标准 - 少量应聘者 |
| 110% (+10%) | 0.4 | 少量 (2-4人/天) | 📝 略高于标准 - 少量应聘者 |
| 115% (+15%) | 0.5 | 一般 (3-5人/天) | 👌 高于标准15%+ - 一般数量应聘者 |
| 125% (+25%) | 0.7 | 较多 (4-7人/天) | ✨ 高于标准25%+ - 较多应聘者 |
| 150% (+50%) | 0.9 | 大量 (6-9人/天) | 💎 远高于标准 - 大量应聘者！ |
| 200%+ (+100%+) | 1.0 | 大量 (7-10人/天) | 💎 远高于标准 - 大量应聘者！ |

## UI实现细节

### 3. 薪资滑块限制

```kotlin
// 滑块范围动态调整为：[最低薪资, 60000]
Slider(
    value = salary.toFloat(),
    onValueChange = { 
        // 确保薪资不低于最低标准
        salary = it.toInt().coerceAtLeast(minSalaryRequired)
    },
    valueRange = minSalaryRequired.toFloat()..60000f,
    steps = ((60000 - minSalaryRequired) / 1000).coerceAtLeast(0)
)
```

## 数值平衡考虑

### 6. 设计理念

1. **硬性标准**：不允许玩家用低薪剥削员工
2. **明确反馈**：清晰的梯度让玩家了解薪资与应聘者的关系
3. **策略选择**：
   - 省钱慢招：刚达标准，慢慢等应聘者
   - 平衡招聘：+15-25%，性价比最佳
   - 快速扩张：+50%以上，大量应聘者快速建队

4. **成本压力**：高级人才（Lv.4-5）最低薪资很高，给玩家带来资金压力

---

## 游戏内使用指南

### 7. 玩家最佳实践

**前期（资金紧张）**：
- 招Lv.1-2员工，薪资设为最低标准的115%（+15%）
- 等待3-5天，会有一般数量应聘者
- 成本可控，慢慢组建基础团队

**中期（稳定发展）**：
- 招Lv.2-3员工，薪资设为最低标准的125%（+25%）
- 每天4-7个应聘者，快速筛选优质人才
- 性价比最佳，推荐策略

**后期（快速扩张）**：
- 招Lv.4-5专家，薪资设为最低标准的150%（+50%）
- 大量应聘者涌入，迅速组建顶级团队
- 成本高但效率最快

## 总结

本次调整实现了一个清晰、合理、有策略深度的招聘薪资系统：

✅ **硬性最低标准**：防止不合理的低薪招聘  
✅ **明确应聘者梯度**：薪资提升带来可预期的应聘者增长  
✅ **策略选择空间**：玩家可以根据财务状况选择不同招聘策略  
✅ **实时UI反馈**：清晰的提示帮助玩家做出决策  
✅ **数值平衡**：既有挑战性又保持合理性  

通过这套系统，玩家将深刻理解"一分钱一分货"的道理，必须支付合理薪资才能吸引到足够的应聘者！
