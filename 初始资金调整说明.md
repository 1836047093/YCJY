# 初始资金调整说明

## 调整内容

将新游戏的初始资金从 **¥100万** 调整为 **¥300万（3M）**。

## 修改详情

**修改文件**：`GameData.kt`

**修改位置**：SaveData 数据类的 money 字段默认值

```kotlin
// 修改前
val money: Long = 1000000L  // 100万

// 修改后
val money: Long = 3000000L  // 300万
```

## 游戏影响

### 前期体验优化

**修改前（100万初始资金）**：
- 招聘2-3名员工后资金紧张
- 需要谨慎控制开支
- 前期压力较大

**修改后（300万初始资金）**：
- 可以招聘更多员工快速组建团队
- 有更多资金进行宣传和运营
- 降低前期经营压力，提升新手体验

### 具体改善

1. **招聘灵活性**：
   - 100万：招聘2-3名Lv.2员工（每人¥20K薪资）
   - 300万：招聘5-8名员工或更高技能员工

2. **开发容错率**：
   - 有足够资金支持多个游戏项目同时开发
   - 可以承受一两个项目的失败

3. **宣传投入**：
   - 可以为游戏投入更多宣传费用
   - 提高首发销量和注册数

4. **收购能力**：
   - 有足够资金收购小型竞争对手（市值200万左右的公司需要240万资金）
   - 更早开始扩张战略

## 游戏平衡

### 破产机制

破产条件未改变：负债达到 **-50万** 时触发破产。

**破产容错**：
- 100万初始 → 破产前可亏损150万
- 300万初始 → 破产前可亏损350万（容错率更高）

### 招聘限制

招聘人数上限仍然基于资金动态调整：
- 资金 ≥ ¥100K → 最多20人
- 资金 ≥ ¥50K → 最多15人
- 资金 ≥ ¥20K → 最多12人

300万初始资金确保玩家可以招募满员。

## 向后兼容

✅ **完全兼容**：
- 旧存档不受影响，继续使用当前资金
- 只有新开档才会使用300万初始资金
- SaveData的money字段只是默认值，不影响已有存档

## 设计理念

1. **降低上手难度**：新手玩家有更多试错空间
2. **加快游戏节奏**：减少前期资金管理压力，更快进入核心玩法
3. **增加策略深度**：有资金后可以尝试更多经营策略（多项目、收购等）
4. **保持挑战性**：后期仍需合理规划，避免盲目扩张导致破产
